# Readoku 📚

AI を活用した英語多読学習支援アプリケーション

ユーザーが入力したプロンプトに基づいて、AI が英語の文章を自動生成し、多読学習をサポートする Web アプリケーションです。

## 💡 開発の背景

英語学習において「多読」は効果的な学習法として知られていますが、以下の課題がありました。

- 自分のレベルや興味に合った教材を見つけるのが難しい
- 市販の教材は内容が限られており、飽きやすい
- 学習の進捗を可視化しにくい

本アプリは、AI を活用してユーザーの興味に基づいた英語コンテンツを無限に生成し、読了記録を管理することで、継続的な多読学習を支援します。

---

## ✨ 主な機能

| 機能            | 説明                                                 |
| --------------- | ---------------------------------------------------- |
| 🔐 ユーザー認証 | サインアップ・ログイン機能（JWT 認証）               |
| 📝 AI 文章生成  | プロンプトに基づいて Gemini API で英語文章を自動生成 |
| 📚 文章管理     | 生成した文章の一覧表示・詳細閲覧・削除               |
| ✅ 読了記録     | 読んだ文章に読了マークを付けて進捗を管理             |
| 📊 統計情報     | 総読了語数・読了回数などの学習統計を表示             |
| 🔄 生成制限     | 1 日あたりの生成回数を制限（コスト管理）             |

## 🚀 環境構築

### 前提条件

- Docker & Docker Compose
- Node.js 20.x（ローカル開発時）
- Go 1.24.x（ローカル開発時）

### 手順

```
# リポジトリをクローン
git clone https://github.com/shuheikomatsuki/readoku.git
cd readoku

# 環境変数ファイルを作成
cp .env.example .env
# .env を編集して必要な値を設定

# コンテナを起動
make up

# ログを確認
make logs
```

## API 一覧

### 認証

| メソッド | エンドポイント   | 説明         |
| -------- | ---------------- | ------------ |
| POST     | `/api/v1/signup` | ユーザー登録 |
| POST     | `/api/v1/login`  | ログイン     |

### ユーザー

| メソッド | エンドポイント                       | 説明         |
| -------- | ------------------------------------ | ------------ |
| GET      | `/api/v1/users/me/stats`             | 学習統計取得 |
| GET      | `/api/v1/users/me/generation-status` | 生成状況取得 |

### 文章（Story）

| メソッド | エンドポイント        | 説明         |
| -------- | --------------------- | ------------ |
| POST     | `/api/v1/stories`     | 文章生成     |
| GET      | `/api/v1/stories`     | 文章一覧取得 |
| GET      | `/api/v1/stories/:id` | 文章詳細取得 |
| PATCH    | `/api/v1/stories/:id` | 文章更新     |
| DELETE   | `/api/v1/stories/:id` | 文章削除     |

## 💪 こだわり・工夫した点

### クリーンアーキテクチャの採用

Handler → Service → Repository の 3 層構造を採用し、責務を明確に分離。

### 生成回数の制限機能

API 利用コスト管理のため、ユーザーごとに 1 日あたりの生成回数を制限。

### 読了記録と統計機能

総読了語数を可視化し、学習モチベーション維持を支援。

### レスポンシブデザイン

モバイル・デスクトップ双方に対応。
